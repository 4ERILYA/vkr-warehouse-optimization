# Методология программной реализации

## 1. Общая схема

Многоуровневая сценарная модель управления запасами реализована
в виде модульного программного компонента на Python. Полный пайплайн:

```
[Данные SKU] → ABC-XYZ-классификация → Кластеризация k-means++ →
              → Назначение CSL (матрица 18 значений) →
              → Расчёт SS, ROP, EOQ → Имитация SimPy → Метрики
```

## 3. Воспроизводимость

Все случайные величины (генерация SKU, симуляция спроса и поставок)
используют явный seed:

- Генерация данных: `seed=2025`
- Имитационные прогоны: `seed = base_seed + run_index * 100`,
  где `base_seed=42`

Это гарантирует, что повторный запуск даст идентичные результаты.

## 4. Сравниваемые политики

| Метка | Описание                                  | CSL       |
|-------|-------------------------------------------|-----------|
| P0    | Единый CSL для всех SKU                   | 0.95      |

| P2    | **CSL по 18 подкластерам (предложенная)** | 0.85–0.99 |
| P3    | Единый высокий CSL                        | 0.99      |

## 5. Метрики оценки

- **Fill Rate** (взвешенный по обороту) — % удовлетворённого спроса
- **Средний запас** в денежном выражении — нагрузка на оборотный капитал
- **TC** — годовые издержки (хранение + заказ + дефицит)
- **Дефицитные циклы** на 100 SKU — сервисная надёжность

## 6. Ожидаемый результат эксперимента

- TC: −7 %
- Дефицитные циклы: −20 %
