## Описание

Имитационная модель управления складскими запасами в стохастических цепях
поставок, объединяющая ABC-XYZ-классификацию, кластеризацию методом k-means++
и расчёт параметров запасов (SS, ROP, EOQ) с дифференцированным целевым
уровнем обслуживания (CSL) на уровне 18 подкластеров.

## Установка

Требования: Python ≥ 3.10.

```bash
pip install -r requirements.txt
```

Зависимости:
- numpy >= 1.24
- scipy >= 1.10
- pandas >= 2.0
- scikit-learn >= 1.3
- simpy >= 4.0
- matplotlib >= 3.7

> **Замечание про SimPy:** если в вашем окружении SimPy не удаётся
> установить (например, в закрытом корпоративном контуре без интернета),
> программа автоматически использует встроенный минимальный fallback из
> `src/_simpy_fallback.py`. Он реализует только то подмножество API,
> которое нужно для модели, и работает идентично основной библиотеке.
> Для полноценного использования рекомендуется поставить настоящий SimPy.

## Запуск

### 1. Полный эксперимент (главное)

```bash
python run_experiment.py
```

Выполняет:
1. Генерация 100 SKU (с фиксированным seed для воспроизводимости)
2. ABC-XYZ-классификация
3. Кластеризация k-means++ внутри каждой группы
4. Расчёт параметров для 4 политик (P0, P1, P2, P3)
5. 30 имитационных прогонов SimPy для каждой политики
6. Сводный отчёт: таблица + графики

Время выполнения: ~3–5 минут.

Результаты сохраняются в `results/`:
- `summary.csv` — сводная таблица сравнения политик
- `all_runs.csv` — детальные данные по всем прогонам
- `comparison.png` — Парето-диаграмма политик
- `groups_effect.png` — эффект P2 над P1 по группам ABC-XYZ
- `inventory_dynamics.png` — динамика запаса для одного SKU

### 2. Базовый пример из статьи (3 SKU)

```bash
python examples/example_3_skus.py
```

Воспроизводит численный пример раздела 3 статьи: расчёт параметров для
SKU-A1 (AX), SKU-B1 (BY), SKU-C1 (CZ).

### 3. Расширенный пример с кластеризацией (6 SKU)

```bash
python examples/example_6_skus.py
```

Демонстрация вторичной кластеризации группы AX на подкластеры α и β
с разной критичностью и сроком поставки.

### 4. Тесты

```bash
python -m pytest tests/ -v
```

Или без pytest:

```bash
python tests/test_components.py
```

## Ожидаемые результаты эксперимента

После запуска `run_experiment.py` получится сводная таблица примерно
такого вида (точные значения зависят от параметров генератора данных):

```
Policy   Fill Rate, %   Inv, k.rub     TC, k.rub      Stockout cycles
P0       99.62 ± 0.12     7640 ± 80       3013 ± 125      5.2 ± 1.3
P1       99.56 ± 0.10     7592 ± 78       3019 ± 103      7.8 ± 1.2
P2       99.70 ± 0.08     7882 ± 88       2926 ± 91       7.3 ± 1.4
P3       99.91 ± 0.07     8705 ± 91       2890 ± 58       1.6 ± 0.7

Преимущество P2 над P1 (предложенная над ABC-XYZ):
  Fill Rate :  +0.14 п.п.
  Запас     :  +3.8 %
  TC        :  -3.1 %
  Дефициты  :  -6.1 %
```

**Ключевой результат:** политика P2 (многоуровневая сценарная) обеспечивает
лучшее сочетание метрик — более высокий Fill Rate при сопоставимом запасе,
а главное — меньшие совокупные годовые издержки (TC) на 3 % относительно
P1 (ABC-XYZ-сегментация). Это и есть главный практический вывод работы:
введение вторичной кластеризации экономически оправдано.

**Замечание о значениях.** Точные значения метрик в эксперименте зависят
от параметров генерируемого набора SKU. В статье и ВКР приведены
теоретически ожидаемые значения, основанные на расчёте через z-фактор
для целевого CSL. На практике имитация даёт несколько более высокий
Fill Rate (поскольку Cycle Service Level и Fill Rate — разные метрики),
но направление и относительная величина эффекта между политиками
сохраняются. См. подробное обсуждение в разделе 4.5 ВКР.

При запуске `run_experiment.py` на консоль выводится:
1. Распределение SKU по группам ABC-XYZ
2. Результаты кластеризации с силуэтными коэффициентами по группам
3. Параметры политики P2 (SS, ROP, EOQ)
4. Прогресс симуляции (4 политики × 30 прогонов)
5. Итоговая сводная таблица сравнения политик
6. Преимущество P2 над P1

В папке `results/` появляются файлы:
- 7 PNG-графиков (Парето-диаграмма, гистограммы, декомпозиция издержек,
  динамика запаса, эффект по группам)
- 3 CSV-таблицы (сводка, все прогоны, параметры SKU)
